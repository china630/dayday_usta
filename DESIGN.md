# 🎨 DayDay Usta - Design System (DESIGN.md)

В этом документе описаны правила пользовательского интерфейса (UI) и пользовательского опыта (UX) для приложения **DayDay Usta**. Любой новый код должен строго следовать этим гайдлайнам.

## 1. Бренд и Логотип
*   **Название:** DayDay Usta
*   **Вайб:** Современный, быстрый, надежный, дружелюбный.
*   **Логотип:** Круглая темная плашка, внутри которой белый силуэт с черными усами и акцентными мятно-зелеными очками/окантовкой. 

## 2. Цветовая палитра (`core/app_colors.dart`)
Дизайн строится на светлом фоне с ярким цветовым акцентом.

| Переменная в коде | Назначение | Визуальное описание | Примечание |
| :--- | :--- | :--- | :--- |
| `kPrimaryColor` | Основной бренд-цвет. Кнопки, шапки, активные табы, иконки. | **Мятно-зеленый (Mint Green / Teal)** | Символизирует скорость, деньги и одобрение. |
| `kBackgroundColor`| Глобальный фон экранов. | **Светло-серый (`#F5F5F5` / `Colors.grey[50]`)** | Используется для контраста с белыми карточками. |
| `Colors.white` | Фон карточек, модальных окон, контейнеров. | **Чисто белый (`#FFFFFF`)** | |
| `kDarkColor` | Основной текст (Заголовки, имена, суммы). | **Темно-серый / Почти черный** | Обеспечивает высокую читаемость. Не использовать абсолютный `#000000`. |
| `Colors.grey` | Второстепенный текст, подсказки, неактивные элементы. | **Серый (`Colors.grey[400]` - `[600]`)** | |

### Семантические цвета (Статусы)
*   🔴 **Красный (`Colors.red`, `Colors.redAccent`):** Отмена заказа, выход из аккаунта, штрафы, срочные вызовы (Təcili).
*   🟠 **Оранжевый (`Colors.orange`):** Статус "Занят" (Sifarişdəsiniz), предупреждения (нет денег на балансе).
*   🟢 **Зеленый (`Colors.green`):** Успешные операции, пополнение баланса, статус "На месте".

## 3. Типографика
Шрифты используются без засечек (Sans-serif). Иерархия размеров жестко зафиксирована:
*   **Крупные заголовки (Шапки, Баланс):** `fontSize: 28` - `32`, `fontWeight: FontWeight.bold`.
*   **Заголовки карточек / Имена:** `fontSize: 20` - `22`, `fontWeight: FontWeight.bold`.
*   **Основной текст / Кнопки:** `fontSize: 16`, `fontWeight: FontWeight.w600` или `bold`.
*   **Мелкий текст / Даты / Субтитры:** `fontSize: 12` - `14`, цвет `Colors.grey`.

## 4. Формы и Скругления (Border Radius)
Приложение использует "мягкий" дизайн без острых углов.
*   **Главные шапки экранов (AppBars/Headers):** Скругление только снизу: `BorderRadius.vertical(bottom: Radius.circular(30))`.
*   **Карточки и Модальные окна (Bottom Sheets):** `BorderRadius.circular(20)`.
*   **Кнопки (ElevatedButton):** `BorderRadius.circular(12)` - `16`.
*   **Табы (TabBar):** `BorderRadius.circular(25)` (стиль "Bubble/Pill").

## 5. Тени и Глубина (Elevation & Shadows)
Используется "плоский" дизайн с очень мягкими тенями для выделения карточек.
*   **Глобальная тень для белых карточек:**
    ```dart
    BoxShadow(
      color: Colors.black.withOpacity(0.05), // Очень слабая тень
      blurRadius: 10,
      offset: const Offset(0, 5)
    )
    ```
*   **Кнопки:** Без теней (`elevation: 0`). Кнопки выделяются только сочным цветом `kPrimaryColor`.

## 6. Паттерны Компонентов (UI Patterns)

### 6.1. Аватарки и Иконки-подложки
Если иконка стоит одна (например, в списке настроек или в карточке услуги), она помещается в круглую подложку с 10% прозрачностью основного цвета:
```dart
Container(
  padding: EdgeInsets.all(8),
  decoration: BoxDecoration(
    color: kPrimaryColor.withOpacity(0.1),
    shape: BoxShape.circle
  ),
  child: Icon(Icons.flash_on, color: kPrimaryColor),
)
```

## 7. Согласованность с PRD (термины и экран поиска)

Документ **[PRD.md](PRD.md)** задаёт продуктовую логику; здесь зафиксированы **те же формулировки** в UI и перекрёстные ссылки.

*   **Типы заказа (пользовательский текст и код):** **Təcili** = срочный (`emergency`), **Planlı** = запланированный (`scheduled`). В русскоязычных комментариях в коде допустимо «срочный / плановый», в интерфейсе — предпочтительно азербайджанские подписи выше (см. PRD §1, §3.3).
*   **Экран ожидания мастера (`pending`):** карточка поверх карты (`OrderTrackingScreen`, `ActiveOrderScreen`) — заголовок **18 pt**, `fontWeight.bold`, цвет **`kDarkColor`**; вторичная строка из **`searchMeta`** (сколько мастеров в зоне, сколько уведомлено, подсказка по радиусу) — **13–14 pt**, серый **`Colors.grey[600]`–`[800]`**, выравнивание по центру (см. PRD §3.3, поле `searchMeta`). При **волне радара свыше 3 км** — дополнительная строка «Axtarış zonası genişləndi» в том же виджете (`OrderSearchMeta.pendingSubtitleAz`, см. **PRD §5.1**). Опционально после мета-текста — **подсказка среднего времени до accept** по категории (`CategoryMetricsService` → `PendingClientSearchSubtitle`, см. **PRD §4.3** `category_metrics`). На **`OrderSearchScreen`** — тот же текст по стриму документа заказа, заголовок **18 pt** bold, субтитр **14 pt** серый (допускается перенос строки), круговой прогресс с **`kPrimaryColor`** (срочный радарный поиск).
*   **Семантика цветов для срочного вызова:** красные акценты допустимы для критичных действий и таймеров (§2); основной прогресс поиска не обязан быть красным, если контекст не отмена — см. экраны `OrderTrackingScreen` / `ActiveOrderScreen` / `OrderSearchScreen`.
*   **Аудит заказа:** события в `orders/{orderId}/events` описаны в **PRD §3.6**; клиент только читает, копирайт экранов истории не противоречит этой модели.
